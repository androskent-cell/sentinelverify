core modules

Seven layers of detection

Each module targets a different forensic dimension of document authenticity.

01 // VISUAL

Visual Fraud Detection

CNN analysis detects pixel-level artifacts — edge distortions, compression irregularities, and GAN/diffusion fingerprints invisible to the human eye.

02 // NLP

Text Pattern Analysis

RoBERTa and BERT models detect linguistic signatures of GPT-4o, Gemini, Llama, and other LLMs — uniform phrasing, high coherence, absence of natural errors.

03 // FORENSICS

Metadata Forensics

Deep audit of hidden file properties: creation timestamps, editing software, author IDs, version history, and digital signatures revealing synthetic tampering.

04 // VERIFY

Cross-Verification Engine

Real-time validation against IRS, OpenCorporates, and state registries to catch phantom entities and invalid EINs embedded in documents.

05 // DASHBOARD

Risk Scoring Dashboard

Composite fraud risk rating synthesizing all detection layers into a single score with annotated previews, audit trails, and exportable PDF reports.

06 // API

REST API & White-Label

RESTful endpoints for integration into banking, insurance, HR, and government digital workflows. White-label options for enterprise clients.

07 // COMPLIANCE

AI Governance Module

Built in alignment with NIST AI RMF, White House AI Bill of Rights, and FedRAMP. Full audit logs and explainability tools included.

detection algorithm

Six-stage pipeline

Every document submitted passes through the full verification pipeline in under 3 seconds.

01 // PREPROCESS

Normalization

Image normalization, OCR extraction, and metadata parsing prepare the document for analysis.

02 // EXTRACT

Feature extraction

CNN models extract visual features. NLP models infer semantic patterns. Metadata classifiers flag anomalies.

03 // ENSEMBLE

Score aggregation

Feature vectors feed an ensemble meta-classifier producing a composite fraud probability.

04 // VERIFY

Registry check

Embedded entity data validated in real time against IRS, OpenCorporates, and state registries.

05 // SCORE

Risk calculation

final_score = 0.7 × fraud_prob + 0.3 × verification_score. Threshold: 0.6.

06 // REPORT

Decision & audit

Authentic or Suspicious verdict with annotated evidence and full exportable audit trail.
